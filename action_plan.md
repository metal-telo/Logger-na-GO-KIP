## Цель
Общая цель проекта – научиться разрабатывать веб-приложения на Go + Angular (?) используя современные подходы.
Данный документ предлагает список задач, решение которых требует использования инструментов, библиотек и подходов, актуальных на момент составления.

По результатам этой работы каждый участник должен научиться на практике использовать современные подходы для своей роли на всех этапах разработки от постановки задачи до поддержки в проде.

## ToDo
Весь список разбивается на разделы: общий для всех и раздельно для разработчиков фронта, бэка и QA.
Также по возможности список ранжируется по важности для изучения.

### Общие

1. Docker. Его использование важно для многих задач разработки и тестирования. Каждый должен уметь собирать/запускать контейнеры.
2. K8s. Де-факто стандарт для развертывания приложений. Каждый должен знать основные сущности, уметь просматривать/редактировать их состояние.
3. Мониторинг/трассировка. Каждый должен знать основные понятия, уметь посмотреть логи/метрики.
4. Использование ИИ для поддержки в решении большинства задач.

### UI Dev

Под задачами UI Dev понимается:

1. Разработка SPA с использованием выбранного фреймворка.
2. Написание юнит-тестов.
3. Настройка пайплайна для сборки (lint, docker).
4. Настройка пайплайна для развертывания (k8s, ingress).
5. Документация по использованию (опционально).

### BE Dev

Под задачами Backend Dev понимается:

1. Разработка нескольких взаимосвязанных сервисов на Go.
2. OpenAPI.
3. Подключение мониторинга, трассировки.
4. Миграция БД.
5. Написание юнит-тестов.
6. Настройка пайплайна для сборки (lint, docker).
7. Настройка пайплайна для развертывания (k8s, ingress).
8. Минималистичная архитектура (DD, seq, adoc?).

### QA

Под задачами QA понимается:

1. Разработка тестовой модели.
2. Написание автотестов на API бэка.
3. Написание заглушек для тестирования.
4. Настройка пайплайна для автоматического запуска тестов.
5. Сборка отчетов о выполненных тестах.
6. Опционально – публикация отчета, отправка уведомлений.
